# GH-001: GitHubとは何か

## これを学ぶ理由

これからAI開発環境やプログラミングを学んでいくと、「GitHub」という名前に何度も出会います。最初にGitHubが何のためのサービスかを知っておくと、この先の教材で迷いにくくなります。

## GitHubとは

GitHub（ギットハブ）は、コードやプロジェクトを**保存し、共有し、他の人と一緒に作業するため**の、クラウドベースのサービスです。

「コードやプロジェクトを保存する」と聞くと、パソコンの中にファイルを保存するのと同じに思うかもしれません。GitHubが違うのは、次のようなことができる点です。

- 自分が作ったものを、他の人に見てもらえる状態で公開・共有できる
- いつ・どこを変更したか、変更の履歴を時系列で追いかけられる
- 他の人に見てもらい、感想や改善案をもらえる
- 複数人で同時に作業していても、お互いの作業が邪魔し合わないように進められる

## Repository（リポジトリ）という言葉

GitHubの中でよく出てくる言葉に **Repository（リポジトリ）** があります。GitHubでは**プロジェクトを置いておく場所**だと考えてください。

「このプロジェクトのリポジトリはこちらです」と言われたら、「このプロジェクトのファイル一式が置かれている場所」という意味になります。まずはこの一言だけ覚えておけば十分です。

## まとめ：次にすること

このレッスンでは、GitHubが「コードやプロジェクトを保存・共有し、みんなで一緒に作業するためのサービス」であることを学びました。

次の教材では、実際にWindowsのブラウザでGitHubの画面を見てみます。今の時点ではアカウント登録やボタン操作は必要ありません。まずは「GitHub = プロジェクトを置いて共有する場所」というイメージを持てていれば準備は完了です。
